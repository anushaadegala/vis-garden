Introduction

The main goal of this project is to visualize the data of the U.S. exports and imports through which any individual can get to know and investigate more about the goods and services that are being shipped from the U.S. to foreign countries also entering the U.S. from foreign countries. As a part of this, we accessed foreign trade data from United States census.gov website. With the prior and extended knowledge on D3.js, we team of 3, have written the code in such a way to get the U.S. state map and other associated charts which are completely interactive with a touch of fun in our project.

Design Process

  1. As a first step in the design process, we understood all the requirements of this project and then planned to come up with a layout of our webpage. Finally after undergoing pros and cons of every design and visualization chart also considering time constraint of this project, we came up with the following layout.
  2. The layout consists of checkboxes for years from 2013-2016 and radio buttons for trading options (imports and exports) followed by countries and commodities. And then we have radio buttons for first layer and second layer. First layer: where we can view small picture of the countries categorized based on region wise in the world , Second layer: where we can view big picture of all the countries in every region..
  3. Then we have a state map with tooltip functionality of the United States including Alaska and Hawaii and excluding territories in the map also in the dataset. Subsequently when a user click on a particular state after choosing the options from checkboxes and radio buttons, a pie chart will be displayed showing either the countries or commodities based on the amount of imports and exports. At the bottom of the page, we have three charts (bar chart, pie chart and line chart) which are interactive with each other which gives more clear cut idea on the U.S. foreign trade from the years 2013-2016.
  4. We kept changing the layout and data structure either because we figured out what we wanted is not logical or new ideas occur to us. For example, about the fun part, we wanted to implement the pac man to, for instance, make it walk through the way from the most amount of exports to the least amount of ones.

Design Choices

  1. About the color: All the colors in this project were used with the help of Color Brewer and tested with Color Oracle. For example, while we are working on the map, we used red color to be filled in the states on mouseover and Green text color for the state code names. But when we checked with color oracle tool if our map even works for people who have colorblindness or not, id did not work for one among three types of colorblindness. Then we tried with other different colors and then came up with #FFE333 which is near to yellow color.
  2. About the data structure: We tried and gave up some structures which We wanted to use for this project, although we had tried it for the four days in a row, including the Fall Break. It isn’t a Fall Break for us, but this project ‘broke our hearts.’ One reason we spent so much time on data structure which we didn’t use eventually is because of the limitation of d3.js. We thought about creating functions which provide data to feed charts and we could use these functions repeatedly. However, we were wrong. The only one way to get data is to retrieve data while building charts. That is to say, we should build data, for example, in an array every time we created charts. That’s probably because of the order of execution of d3.js. Moreover, another thing we tried for fetching data from csv files is we reduced data, leaving only 3 states, in order to valid if we get the right number from those files.
